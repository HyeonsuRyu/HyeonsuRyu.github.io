Java 이해하기

Java

 나는 C언어를 통해 프로그래밍을 처음 접하고 이후 Python을 공부했었다. C언어를 이해한 후 Python을 공부할 땐 문법이나 여러 기능면에서 차이가 있지만 Python 프로그래밍의 구조 이해가 어렵진 않았다. 그러나 이후 Java를 공부하면서 project package class java public static 등등 처음보는 개념이 너무나 많았고 큰 어려움을 겪었다. 사실 이런 것들은 예제를 따라해보며 Java에 대한 이해를 어느정도 한 후에 이해해도 상관 없는 내용이다. 하지만 모르는 것이 있는데 넘어가는 걸 좋아하는 성격이 아니여서 하나 하나 찾아보았다.

Java 프로젝트 구조

  • ProjectA
    • packageA
      • ClassA
      • ClassB
    • packageB
      • ClassB

 eclipse, Intellij 같은 IDE들은 폴더 경로를 이용하여 전체 프로젝트를 관리한다. 이 글은 intellij를 기준으로 한다. 이때 workspace project package class의 개념이 나온다. IDE를 통해 Java를 시작하려면 먼저 project를 만들게 된다. project를 만들면 해당 폴더 안에 .idea폴더와 src폴더가 생성된다. .idea 폴더는 Intellij에서 Java 프로젝트를 관리하는 정보가 있는 폴더이고 프로그래밍은 src폴더에서 이뤄진다. Java 프로그램을 만들기 위해서는 이 src폴더에 .java 확장자를 가진 class 만들어야한다. 자바에서 프로젝트를 진행하면 class를 여러개 만들게 되는데 이 class들을 정리하기 위하여 package를 이용한다. package는 주로 비슷한 역할을 지닌 class들을 묶어서 보관한다. 이때 package가 다르다면 class 이름이 중복 돼도 된다. + method는 파일 경로를 통해 구분되는 것은 아니지만 class 안에서 생성된 함수이다.

프로젝트 명명법

 프로젝트는 파스칼 표기법을 사용한다.

패키지 명명법

 패키지 이름은 모두 소문자로 표기하는걸 권장하고, 가독성을 위해 대문자를 섞기도 한다. 패키지 이름을 만들 땐 한가지 더 신경써야한다. 단순 공부용으론 필요 없지만 여러 사람과 협업하거나 프로젝트가 커질경우 패키지명이 중복되는 것을 피해야한다. 때문에 고유한 이름인 회사나 개인의 홈페이지 도메인을 이용하여 도메인.필요한 정보1.필요한 정보2 와 같이 명명한다. 필요한 정보에는 작성자 이름 등이 있을 수 있다. 이때 도메인은 순서를 바꿔 이용한다. google.com을 예로 들면, com.google과 같은 형식이다.

클래스 명명법

 클래스는 카멜 표기법을 사용하는 것이 권장되지만 2번째 이후 단어는 소문자로 시작하기동한다. 이때 첫 단어는 명사로 만들어 같은 표기법을 이용하는 메소드와 구분되게 한다.

메소드 명명법

 메소드는 클래스와 같은 표기법을 사용하고 동사로 시작해 클래스와 구분되게 한다.

Java 실행 과정

.java - -compiler-> class - -jvm-> 실행

 Java를 쓰는 이유는 높은 이식성이 있다. 다른 언어들은 플랫폼이 달라지면 코드를 다시 작성해야 되지만 Java는 그럴 필요가 없다. Java로 프로그래밍 하면 .java 확장자 파일을 가진 소스파일을 작성하게 된다. 이 코드를 컴파일 하게 되면 컴파일러는 .java 파일을 class 파일로 바꾸게 된다. 이 class 파일은 다시 JVM이라는 자바 가상 머신에서 실행되게 된다.  C언어로 프로그래밍 하면 .c파일은 바로 실행 가능한 .exe 파일로 바뀌게 된다. 이에 비하면 java는 과정을 하나 더 거치게 되고 결과적으로 실행속도를 늦추게 된다. 그러나 JVM만 있다면 어떤 운영 체제에서든 실행 가능하게 돼 Java에서는 exe파일을 생성하는 것이 아니라 class파일을 만들고 JVM을 통해 실행하는 것이다.

Hello World 뜯어보기

public class HelloWorld {
  public static void main(String[] args) {
    System.out.println("Hello World!");
  }
}

 위 코드는 Hello World!를 출력하기 위한 Java 코드이다. HelloWorld 하나 출력을 위해 public, class, static, args같은 것들이 있다. IDE를 사용하면 class 선언은 자동으로 되고 main 메소드도 ‘psvm’ + enter면 해결된다. 때문에 println을 이해하기 위해 처음 Java를 공부하는 사람이 이 모든것을 알 필요는 없지만 찝찝한 마음에 조사해보았다.

public

 public은 접근 제한자이다. 말 그대로 접근할 수 있는 범위를 제한하는 것이다. public은 다른 클래스에서도 사용할 수 있음을 의미한다. public 외의 접근 제한자로는 private, protected가 있다.

static

  static은 정적이라는 의미로 메소드나 변수에 사용할 수 있다. static이 없으면 인스턴스로 생성된다.

 먼저 메소드는 static을 지정해주면 인스턴스를 따로 생성하지 않아도 사용 가능하고, static이 없다면 인스턴스를 사용할 곳에서 생성해야한다.

class getNumber{
    static int getNumberA() {
      return 10;
    }

    int getNumberB() {
      return 5;
    }
}

public class exampleStaticMethod {

    public static void main(String[] args) {
        System.out.println(getNumberA);

        getNumber getNum = new getNumber();
        System.out.println(getNum.getNumberB);
    }
}

 변수 static을 사용하면 여러 객체에서 같은 메모리를 사용하고 static이 없으면 인스턴스를 따로 생성해 사용한다. 따라서 변수값을 수정해도 다른 객체에는 영향을 미치지 않는다.

class Number{
    static int num1 = 10;
    int num2 = 5;
}

public class exampleStaticField {

    public static void main(String[] args) {
    	Number numA = new Number();
    	Number numB = new Number();

    	numA.num1 = 100;
    	numA.num2 = 200;
    	System.out.printf("%d, %d", numB.num1, numB.num2);
    }
}

 위 코드에서 인스턴스로 생성된 numA를 통해 Number 클래스의 num1과 num2를 수정했다. 이때 num1은 정적으로 생성된 변수여서 numA와 numB 인스턴스가 같은 메모리를 사용해 numB에서도 수정되었고 num2는 정적이 아니여서 수정되지 않았다. 따라서 출력은 100, 5이다.

String[] args

 String[] args는 args라는 변수명을 가진 문자열 데이터형의 배열을 의미한다. 커멘드에서 자바를 실행할 때 넘겨주는 값을 main으로 가져오게된다.
